# Архитектура плагина LogCat

## 1. Обзор
Плагин предназначен для альтернативного отображения логов Android LogCat с расширенными возможностями фильтрации по тегам.

## 2. Основные компоненты

### 2.1. UI Layer (ToolWindow)
- **LogCatToolWindowFactory**: Точка входа для создания ToolWindow.
- **LogCatPanel**: Основная панель с текстовым полем (ConsoleView) и панелью инструментов (Toolbar).
- **TagFilterDialog**: Модальное окно для выбора тегов.

### 2.2. Data Layer (LogCat Integration)
- **LogCatListener**: Компонент, подписывающийся на поток логов через `AndroidLogcatService` или аналогичные механизмы IntelliJ SDK.
- **LogBuffer**: Буфер для хранения входящих сообщений до применения фильтров.

### 2.3. Filtering Logic
- **TagManager**: Собирает уникальные теги из потока логов. Хранит состояние выбранных тегов.
- **LogFilterModel**: Применяет выбранные фильтры к потоку данных перед отображением в UI.

### 2.4. Storage (Settings & State)
- **LogCatSettings**: Хранит настройки отображения (фонты, цвета).
- **FilterState**: Хранит список выбранных тегов для восстановления после перезапуска.

## 3. Технологический стек
- **IntelliJ Platform SDK**: `com.intellij.modules.androidstudio`
- **Kotlin**: 2.1.0
- **UI**: Swing (стандарт для плагинов IntelliJ)

## 4. Сценарии взаимодействия

### 4.1. Перехват логов
1. При активации плагина инициализируется `AndroidLogcatService`.
2. Подписка на сообщения для выбранного устройства/процесса.
3. Каждое сообщение передается в `TagManager` для обновления списка тегов и в `LogFilterModel`.

### 4.2. Фильтрация по тегам
1. Пользователь нажимает "Фильтр".
2. `TagFilterDialog` получает список тегов из `TagManager`.
3. После подтверждения `LogFilterModel` обновляет представление в `ConsoleView`.
